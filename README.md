## 📋 Описание

Автоматизированный bash-скрипт для:
- 🔍 Поиска Wi-Fi сетей с включенным и незаблокированным WPS
- 🎯 Автоматического выбора целей на основе качества сигнала
- ⚡ Атаки с использованием метода Pixie Dust (через oneshot.py)
- 📊 Логирования результатов и сохранения найденных паролей

## ✨ Особенности

- ✅ **Автоматический поиск** сетей с Locked=No
- ✅ **Сортировка по сигналу** - самые сильные сети в приоритете
- ✅ **Режимы атаки**:
  - Атака выбранной сети
  - Автоматическая атака всех сетей
  - Атака только сетей с хорошим сигналом
- ✅ **Контроль времени** - таймауты для зависших атак
- ✅ **Live-вывод** - видите всё, что делает oneshot.py в реальном времени
- ✅ **Повторные попытки** - автоматические ретраи при неудаче
- ✅ **Сохранение результатов** - найденные пароли в отдельный файл
- ✅ **Красивый цветной вывод** в терминале

## 📦 Требования

### Установленные пакеты
```bash

```

### Необходимые файлы
- `oneshot.py` - должен находиться в той же директории (или указать путь через `-p`)

### Поддерживаемые Wi-Fi адаптеры
Адаптеры с поддержкой режима монитора (на чипах):
- Atheros (AR9271, AR9287 и др.)
- Ralink (RT3070, RT5370)
- Realtek (RTL8187, RTL8812AU)
- Некоторые другие

```bash
chmod +x wps_auto.sh

sudo apt update

```

## 📖 Использование

### Базовый запуск
```bash
sudo ./wps_auto.sh
```

### С дополнительными параметрами
```bash
# Указать другой Wi-Fi интерфейс
sudo ./wps_auto.sh -i wlan1

# Увеличить таймаут до 5 минут и 3 попытки
sudo ./wps_auto.sh -t 300 -r 3

# Указать путь к oneshot.py
sudo ./wps_auto.sh -p /opt/tools/oneshot.py

# Комбинация опций
sudo ./wps_auto.sh -i wlan0 -t 240 -r 2
```

### Параметры командной строки

| Параметр | Описание | По умолчанию |
|----------|----------|--------------|
| `-i, --interface` | Wi-Fi интерфейс | `wlan0` |
| `-t, --timeout` | Таймаут атаки (секунды) | `180` |
| `-r, --retries` | Количество попыток | `2` |
| `-p, --oneshot-path` | Путь к oneshot.py | `./oneshot.py` |
| `-h, --help` | Показать справку | - |

## 🔧 Устранение неполадок

### Ошибка: "wash: command not found"
```bash
sudo apt install reaver -y
```

### Ошибка: "No wireless interfaces found"
- Проверьте, что Wi-Fi адаптер подключен
- Проверьте драйверы: `iwconfig`
- Убедитесь, что адаптер поддерживает режим монитора

### Атака зависает на "Authenticating"
Скрипт автоматически:
- Определяет зацикливание через таймаут
- Переходит к следующей сети
- Делает повторные попытки

### Слабый сигнал
- Используйте адаптер с внешней антенной
- Подойдите ближе к точке доступа
- Выберите опцию `G` (только хороший сигнал)

## 📊 Рекомендации по использованию

1. **Для домашнего тестирования:**
   - Используйте адаптер с поддержкой monitor mode
   - Находитесь в зоне уверенного сигнала (RSSI >= -65)
   - Настройте таймаут 180-240 секунд

2. **Для тестирования нескольких сетей:**
   - Используйте режим `A` (все сети)
   - Установите разумный таймаут (120-180 сек)
   - Скрипт автоматически перейдет к следующей сети

3. **Проверка безопасности своей сети:**
   - Запустите скрипт против своего роутера
   - Если WPS взломан - **ОТКЛЮЧИТЕ WPS** в настройках!

## 🛡️ Как защититься от таких атак

1. ✅ **ОТКЛЮЧИТЕ WPS** в настройках роутера (самое важное!)
2. ✅ Используйте сложный пароль (не менее 12 символов)
3. ✅ Включите WPA2/WPA3 шифрование
4. ✅ Регулярно обновляйте прошивку роутера
5. ✅ Отключите WPS PIN (оставьте только кнопочный, если нужен)
